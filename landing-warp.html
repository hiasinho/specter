Sync

Push. Pull. Stay current.

Specter syncs your markdown collection against a shared remote. Every push computes SHA256 hashes so only what changed travels over the wire. Pull with a single command to get your team's latest.

  • Hash-based diff — only changed files sync
  • Revision tracking via .specter-sync
  • Preview changes before they land with status and diff

Conflict detection

Conflicts surface before they hurt.

Every push checks the remote revision against your last-synced state. If someone else updated a document you also touched, Specter stops and tells you exactly which files diverged — before anything overwrites anything.

  • SHA256 content hashing — no false positives
  • specter diff shows exactly what changed, document by document
  • Never silently overwrites remote work
Proposals

Suggest changes.
Review before merging.

Specter proposals are lightweight change requests for documents — like pull requests, but for your specs. Propose edits from a branch, list open proposals, and approve or reject them inline from the CLI.

  • specter propose opens a proposal from current changes
  • specter proposals lists all open proposals
  • specter review approve or reject with a comment

→ specter proposals #3 Update auth spec alice · 2h ago → docs/auth.md (2 changes) #4 Add deployment runbook bob · 45m ago → docs/runbook.md (new file) → specter review 4 --approve ✓ Proposal #4 approved and merged

Branch-aware

Your git branch is your namespace.

Specter reads your current git branch and scopes every push and pull to that branch automatically. Work on feature/redesign-api without touching main.

  • Auto-detected from git rev-parse — zero config
  • Isolated revisions per branch
  • Merge work back to main via proposals
Quickstart

Up and running in two minutes.

1 · Install
$ go install github.com/hiasinho/specter@latest
2 · Authenticate
$ export SPECTER_TOKEN=your-token
3 · Init your project
$ specter init
Created .specter — edit project and paths to configure
4 · Sync
$ specter push
✓ Synced 4 documents to revision r1
