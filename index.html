Specter

A sync layer for your team's documents, built for AI agents and humans alike.

Keep specs, docs, and decisions in sync across repos and teams. Push from your repo, collaborate through agents or the API.

How it works

01

Configure

Drop a .specter file in your repo to define which folders to sync. 

project: my-project
paths:
  - specs/
  - docs/architecture/
02

Push

Your documents land on Specter, versioned by branch. The CLI auto-detects your git branch. 

$ specter push
Pushing 12 documents to main...
✓ Synced in 340ms
03

Collaborate

Teammates and AI agents read, review, and propose changes through the API. 

POST /projects/my-project/proposals
{
  "document": "specs/auth.md",
  "type": "replace",
  "body": "..."
}

What it does

Document sync

Push markdown from any repo, pull updates back. Content hashing and revisions keep everything in order.

Branch-aware

Documents follow your git branches, with automatic fallback to main when a branch doesn't exist on the service.

Proposals

Structured feedback — replace, insert, delete, or annotate. Anchored to content, not line numbers.

Agent-friendly API

RESTful, token-based, designed for LLM agents to read documents and submit proposals programmatically.

Role-based access

Owner, editor, reviewer, reader. Invite collaborators with scoped permissions via shareable invite codes.

No infrastructure

Runs on Supabase. No servers to manage, no deploy pipelines. Push your docs and go.

Status

Specter is under active development. The API and core sync are functional, but things will change.
