Features

Everything your docs need.
Nothing they don't.

Push & Pull.

Sync your markdown documents with a single command. Specter handles conflict detection via SHA256 hashing — only diffs travel over the wire.

specter push specter pull
Proposals.

Propose changes without overwriting. Teammates review diffs inline, approve or reject. It's a pull request — for your living docs.

specter propose specter review
Conflict Detection.

Every push is compared against the last known revision. If someone else pushed first, Specter catches it — before you overwrite their work.

specter status specter diff
Branch-Aware.

Specter reads your current git branch and namespaces your documents automatically. Work on feature/new-api without touching main.

specter history specter invite
CLI Reference

One binary.
All you need.

Specter is a single static binary. Drop it in your PATH, set SPECTER_TOKEN, and you're running. No daemon, no background service.

push Upload local docs to remote, update revision
pull Download latest remote docs to local filesystem
status Show sync state and detect local vs remote drift
diff Show line-by-line diff against remote revision
propose Create a change proposal for review
review Accept or reject pending proposals
history List document revisions, view or diff any two
invite Add team members to your project
init Scaffold .specter config in current repo

Config

Zero boilerplate.

Drop a .specter file at your repo root. Point it at your docs directories. That's it. Branch namespacing and conflict tracking are automatic.

Auth via SPECTER_TOKEN env var
Git branch auto-detected
.specter-sync gitignored automatically

project: acme/backend-docs
paths:
- docs/
- specs/
exclude:
- **/drafts/**
