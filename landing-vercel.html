Core capabilities

Push / Pull

Push your docs.
Pull what changed.

Sync entire document collections with a single command. The CLI collects markdown files from your configured paths, hashes each one, and only transmits what changed.

  • Branch-namespaced — each git branch gets its own state
  • --force flag to bypass conflict guard when you mean it
  • Sync state stored in .specter-sync (gitignored)
terminal 
$ specter push

Pushed to hiasinho/specter/main:
  created: 2
    + specs/api-reference.md
    + specs/auth-model.md
  updated: 1
    ~ specs/overview.md
  unchanged: 4

$ specter push  # conflict detected

Push rejected: 1 document(s) modified on server since last pull:
  ! specs/api-reference.md (server revision 5)

Run `specter pull` to sync, then push again.
.specter 
# drop this file at your repo root

project: hiasinho/my-project

paths:
  - specs/
  - docs/architecture/
  - docs/api/

exclude:
  - drafts/
  - "*.wip.md"

# auth via env — never committed
# export SPECTER_TOKEN=your-token
Zero boilerplate

One config file.
That's all.

A single .specter YAML at your repo root. Declare which paths to sync and what to exclude. The CLI walks up the directory tree to find it — no matter where you run from.

  • Project namespace: owner/slug format
  • Glob exclusion patterns supported
  • Auth token via SPECTER_TOKEN env — never in config
Proposals

Suggest edits
without overwriting.

Reviewers propose targeted changes — replace, insert, delete, or annotate — anchored to exact text positions. Editors accept or reject. Nothing mutates until approved.

  • Types: replace insert delete note
  • Anchored by content snippet + line hint
  • Designed for AI agents operating as reviewers
specter propose 
$ specter propose specs/auth-model.md \
    --type replace \
    --anchor "Users authenticate via JWT" \
    --body "Users authenticate via API tokens"

Proposal created: #a3f9c2d
  document:  specs/auth-model.md
  type:      replace
  status:    pending

$ specter proposals # list open

#a3f9c2d  specs/auth-model.md  replace  pending
#b1e44f1  specs/overview.md    note     pending

$ specter review a3f9c2d --accept

History

Every revision.
Instantly diffable.

Specter retains the last 20 revisions per document. View the full edit trail, inspect any past version, and diff two revisions with unified output — just like git diff.

  • Author attribution on every revision
  • Unified diff format — pipe anywhere
  • specter diff compares local against remote

REST API

Built for
AI agents.

Every CLI command maps to a clean REST endpoint. Your agents can push context, pull the latest specs, and raise proposals — programmatically, without the CLI.

POST /sync/:owner/:slug bulk push
GET /sync/:owner/:slug pull changes
POST /proposals/:owner/:slug create proposal
PATCH /proposals/:owner/:slug/:id accept / reject
GET /document-history/:owner/:slug/:path/diff unified diff

Quickstart

Get started in seconds.

No daemon. No cloud config UI. Just a binary and a token.

1 Install the CLI 
$ go install github.com/hiasinho/specter@latest
2 Register and set your token 
$ specter register --email you@example.com --username yourname \
    --invite-code a1b2c3d4e5f6a7b8

Registered. Your token: 4e9f2a...

$ export SPECTER_TOKEN=4e9f2a...
3 Initialize a project 
$ specter init --name "My Project" --slug my-project

Project created: yourname/my-project
Config written to .specter
4 Push your documents 
$ specter push

Pushed to yourname/my-project/main:
  + specs/overview.md
  + specs/api.md
5 Invite your team (optional) 
$ specter invite --role editor

Invite created. Share this code: f7c91e44...

# Teammate redeems it:
$ specter invite redeem f7c91e44

Need an invite code to register? Specter is invite-only during early access.

All commands

specter push

Sync local markdown files to the server. Conflict-safe by default.

specter pull

Pull documents changed since last sync. Updates local files.

specter status

Show which local documents differ from the remote.

specter diff

Show unified diff between local and remote content.

specter propose

Submit a targeted proposal to change a document.

specter proposals

List all pending and resolved proposals on a project.

specter review

Accept or reject a proposal by ID.

specter history

View revision history for a document. Show any past version.

specter invite

Generate or redeem an invite link with a specific role.

specter init

Create a project and write .specter config.

specter register

Create a user account with an invite code.

specter skill

Fetch and print the AI agent skill definition for this project.
