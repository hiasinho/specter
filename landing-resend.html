Push & Pull

One command.
Everything synced.

Push your entire document tree in a single invocation. Pull only what changed since your last sync. Branch-namespaced — your feature branch stays isolated.

  • → Reads from configured paths in .specter
  • → Glob-based exclusion patterns
  • → Tracks revision number in .specter-sync
.specter
# project config
project: acme/specs
paths:
- specs/
- context/
exclude:
- drafts/**
- **/*.tmp.md
$ specter push
↑ 5 pushed · 2 unchanged
$ specter pull
↓ 1 updated (specs/api.md)

Conflict detection

SHA-256 truth,
no guessing.

Every push is compared against the server's last-known revision. If another writer got there first, the push is rejected cleanly — not silently overwritten.

  • → Content-hash comparison (SHA-256)
  • → 409 conflict response with per-file detail
  • → specter diff to inspect before resolving

Proposals

Propose changes.
Not rewrites.

Reviewers submit targeted proposals — replace, insert, delete, or annotate — anchored to exact content. Editors accept or reject without leaving the terminal.

  • → Four proposal types: replace, insert, delete, note
  • → Anchor-based targeting — no line numbers that drift
  • → Role-gated: reviewer submits, editor decides
terminal — proposal workflow
$ specter propose specs/api.md
Type: replace
Anchor: "Authentication requires..."
Body: "All requests require x-specter-token"
✓ Proposal #7 submitted
$ specter proposals --status pending
#7 replace specs/api.md reviewer:alice
#6 note specs/auth.md reviewer:bob
$ specter review 7 --accept
✓ Proposal #7 accepted

History

Every revision,
diffable.

Specter keeps the last 20 revisions per document. Browse the log, inspect any version, or diff two revisions — unified diff format, same as git.

  • → specter history <file> — list revisions
  • → specter history <file> --rev 5 — show content
  • → Author, timestamp, hash per revision
